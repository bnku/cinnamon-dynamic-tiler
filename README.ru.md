# Dynamic Tiler 🚀

[English](README.md) | Русский

**Dynamic Tiler** — это настраиваемое расширение для Cinnamon, реализующее концепцию «умного» тайлинга окон. Оно объединяет гибкость традиционного плавающего рабочего стола и высокую производительность тайловых менеджеров. Сочетая независимое, покрытое тестами математическое ядро на TypeScript с нативным расширением на GJS (Gnome JavaScript), проект обеспечивает предсказуемое управление окнами с клавиатуры, перетаскивание мышью с понятной визуальной подсветкой и ведение истории транзакций для безопасного отката изменений.

Вместо того чтобы принудительно втискивать окна в жесткую статичную сетку, которая ломается при обычном повседневном использовании десктопа, Dynamic Tiler адаптируется под ваши действия. Он сдвигает, сжимает и восстанавливает размеры окон динамически, предлагая эффективность тайлинга и сохраняя при этом привычное удобство плавающих окон.

## 📖 История проекта и мотивация

Большинство разработчиков и опытных пользователей ценят саму идею тайлинга: эффективное использование площади экрана, быстрое управление с клавиатуры и аккуратное расположение окон. Однако на практике классические тайлеры (такие как i3, dwm или bspwm) часто приносят с собой неприятные неудобства:
* Обычные плавающие окна — диалоги настроек, утилиты, палитры цветов — растягиваются на пол-экрана, ломая всю структуру.
* Работа с мышью либо отключена, либо реализована неудобно.
* Случайное перетаскивание или изменение размеров окна может безвозвратно разрушить тщательно выстроенную мозаику рабочего пространства.
* Настройка и перенос конфигураций между разными средами рабочего стола часто превращаются в сложную ручную работу.

Dynamic Tiler создавался как попытка решить эти проблемы напрямую. Проект начинался как экспериментальный скрипт для X11, который управлял окнами извне с помощью CLI-утилит (`wmctrl`, `xdotool`). Хотя математика работала правильно, запуск внешних процессов на каждое нажатие клавиши создавал заметную задержку (~150 мс и более).

Чтобы сделать отклик мгновенным и плавным, мы реорганизовали кодовую базу:
1. **Отделение математического ядра:** Мы перенесли всю геометрию сетки и алгоритмы разрешения конфликтов в чистое TypeScript-ядро, полностью независимое от API операционных систем и оконных менеджеров.
2. **Нативная интеграция с Cinnamon:** Мы интегрировали это ядро непосредственно в нативное расширение Cinnamon Spice. Оно выполняется прямо в потоке рендеринга Mutter/Muffin, что исключило накладные расходы на запуск процессов и снизило время отклика до незаметных глазу **8–12 мс**.
3. **Фокус на UX:** В ходе разработки мы добавили надежные защитные механизмы: транзакционный откат истории, эластичный ресайз («гармошку»), гистерезис курсора для предотвращения мерцания подсветки и автоматический выбор оси нарезания сетки.

Сегодня Dynamic Tiler — это практичный и удобный инструмент управления окнами, созданный для комфортной и предсказуемой ежедневной работы.

## 🚀 Начало работы

### 📋 Требования
* Linux Mint / Cinnamon Desktop Environment (на X11).
* Node.js и npm.
* Только для устаревшего CLI: `wmctrl`, `xdotool`, `x11-utils` и `x11-xserver-utils`.

Установка системных пакетов в Ubuntu / Linux Mint:
```bash
sudo apt update
sudo apt install nodejs npm wmctrl xdotool x11-utils x11-xserver-utils
```

### 📦 Установка
1. Склонируйте репозиторий.
2. Установите зависимости разработки и запустите тесты для проверки ядра:
   ```bash
   npm install
   npm test
   ```
3. Скомпилируйте TypeScript код и установите расширение в локальную директорию Cinnamon одной командой:
   ```bash
   npm run build:extension
   ```
   *(Команда соберет проект с помощью `esbuild` и скопирует бандл напрямую в каталог `~/.local/share/cinnamon/extensions/dynamic-tiler@cinnamon.org/`)*
4. Перезагрузите Cinnamon (нажмите `Alt + F2`, введите `r` и нажмите `Enter`) либо откройте приложение **Cinnamon Extensions** и активируйте **Dynamic Tiler**.

## ⌨️ Руководство пользователя: Горячие клавиши

Клавиатурные сочетания Dynamic Tiler спроектированы так, чтобы управление было интуитивным и отзывчивым. Ниже приведена таблица хоткеев по умолчанию и описание того, как они ведут себя на практике.

### Горячие клавиши и модификаторы (Настраиваемые параметры)

| Действие | Комбинация по умолчанию | Ключ в файле настроек | Результат / Ощущение в UX |
| :--- | :--- | :--- | :--- |
| **Tile Left** | `Super + Left` | `keybinding-tile-left` | Расширяет или притягивает окно влево, сдвигая соседей. |
| **Tile Right** | `Super + Right` | `keybinding-tile-right` | Расширяет или притягивает окно вправо, сдвигая соседей. |
| **Tile Up** | `Super + Up` | `keybinding-tile-up` | Расширяет или притягивает окно вверх, сдвигая соседей. |
| **Tile Down** | `Super + Down` | `keybinding-tile-down` | Расширяет или притягивает окно вниз, сдвигая соседей. |
| **Shift Left** | `Ctrl + Super + Left` | `keybinding-shift-left` | Моментально перемещает («перебрасывает») окно к левому краю. |
| **Shift Right** | `Ctrl + Super + Right` | `keybinding-shift-right` | Моментально перемещает («перебрасывает») окно к правому краю. |
| **Shift Up** | `Ctrl + Super + Up` | `keybinding-shift-up` | Моментально перемещает («перебрасывает») окно к верхнему краю. |
| **Shift Down** | `Ctrl + Super + Down` | `keybinding-shift-down` | Моментально перемещает («перебрасывает») окно к нижнему краю. |
| **Restore Window** | `Super + BackSpace` | `keybinding-restore` | Выводит окно из сетки, восстанавливая плавающий размер. |
| **Модификатор Snap (DND)** | `Ctrl` (удержание при drag) | `dnd-modifier-key` | Примагничивает перетаскиваемое окно к сетке (синяя подсветка). |
| **Модификатор Swap (DND)** | `Ctrl + Shift` (удержание) | `dnd-swap-modifier-key` | Меняет два окна физической геометрией местами (зеленая рамка). |

### Как хоткеи ощущаются на практике

* **Tile Left / Right / Up / Down:** 
  - Если активное окно является плавающим, нажатие этих клавиш мгновенно примагничивает его к соответствующей половине экрана.
  - Если окно уже находится в сетке, нажатие той же стрелки увеличивает его границу на шаг сетки (`step`), плавно сдвигая соседние окна.
  - Если окно упирается в край экрана, оно сожмет соседей до их минимального размера (`minColumnSpan`/`minRowSpan`). Если сжимать соседей больше нельзя, активное окно начнет уменьшаться с *противоположной* стороны, позволяя соседнему окну расшириться. Это исключает «мертвые зоны» и создает ощущение, что окна плавно скользят по рельсам.
* **Shift Left / Right / Up / Down:** 
  - Это жест быстрого перемещения. Вместо последовательного изменения размеров по шагам сетки активное окно сразу отправляется в крайнюю половину сетки в указанном направлении. Удобно, когда нужно быстро убрать терминал или чат к краю экрана.
* **Restore Window:** 
  - Это ваш быстрый выход из тайлинга. Если вам временно понадобилось поработать с окном в свободном плавающем режиме, это сочетание клавиш мгновенно возвращает его к исходному виду. Одновременно с этим движок обнаруживает пустующее место и аккуратно схлопывает вакансию, заставляя окружающие окна занять свободную область без образования щелей.

## 🖱️ Руководство пользователя: Управление мышью

Dynamic Tiler предлагает интуитивный рабочий процесс перетаскивания окон с привязкой к сетке.

* **Прилипание к сетке:** Зажмите клавишу-модификатор (по умолчанию `Ctrl`, настраивается как `<Control>d` в настройках) во время перетаскивания окна. Появится полупрозрачная синяя рамка превью, показывающая будущую позицию.
* **Вставка в стек (Stack Insertion):** Перетащите окно на стык между двумя окнами в вертикальной или горизонтальной колонке. Превью покажет, что для него открывается новая строка/колонка посередине.
* **Проскальзывание по коридору:** Подтяните окно к краю экрана мимо существующего окна. Соседи сдвинутся внутрь, освобождая коридор под курсором.
* **Явный обмен (Swap):** Перетащите одно окно поверх другого с зажатыми клавишами `Control + Shift` (по умолчанию, настраивается как `<Control><Shift>d`). Превью сменит цвет на зеленый. Отпустите мышь — окна поменяются местами.
* **Вытаскивание из сетки:** Просто начните тащить заголовок окна с зажатым модификатором в сторону от его сетки. На месте окна появится очертание пустующей вакансии. Если увести мышь дальше 80 пикселей и отпустить — вакансия схлопнется, а соседи вернут прежние формы.

## ⚙️ Настройки и кастомизация

Интерфейс настроек встроен нативно в окно управления расширениями Cinnamon и разделен на четыре вкладки:

1. **Layout (Раскладка):**
   * **Gaps:** расстояние между окнами в пикселях (например, `8px` для аккуратных отступов).
   * **Enable Previews:** включение полупрозрачных превью перед сбросом окна.
   * **Grid Columns / Rows:** настройка базовой плотности сетки (например, `12 x 6` или `8 x 6`).
   * **Monitor Profiles & Overrides:** возможность задать индивидуальные сетки для горизонтальных, вертикальных или ultrawide экранов, либо прописать точечные переопределения вручную (например, `0:24x12, 1:6x12`).
   * **Minimum Column / Row Spans:** ограничение минимальной ширины и высоты окна в ячейках сетки.
2. **Drag & Drop:**
   * Включение мышиного тайлинга и настройка клавиши-модификатора (по умолчанию `<Control>d`).
   * Включение обмена окон (Swap) и настройка его клавиш (по умолчанию `<Control><Shift>d`).
3. **Keyboard:**
   * Привязка хоткеев для ресайза во всех направлениях, сдвига окон к краям и восстановления геометрии.
4. **Diagnostics:**
   * Включение записи отладочной информации и трассировки решений в системный лог `~/.xsession-errors`.

## 🏗️ Под капотом: Модульное ядро (Для разработчиков)

Кодовая база собрана вокруг небольшого TypeScript-ядра, не завязанного на конкретный оконный менеджер. Математика раскладки живет отдельно от Cinnamon/X11-интеграции, а адаптеры отвечают за API оконного менеджера, настройки, кэш и детали конкретной среды.

Поскольку вся математика тайлинга полностью изолирована от системных API, само ядро можно перенести в любую другую среду (например, GNOME Shell, KDE KWin или Wayland-композиторы на wlroots) — для этого достаточно написать новые адаптеры инфраструктуры.

### Назначение модулей ядра:
* **`GridSpans.ts`**: Работает с логическими индексами колонок и строк активной сетки.
* **`GeometryConverter.ts`**: Преобразует физические координаты пикселей в логические спаны сетки и обратно, учитывая отступы (gaps), рамки окон и границы рабочих зон монитора.
* **`InitialLayout.ts`**: Ищет свободные зоны при появлении нового окна, аккуратно размещая его с краю или рядом с активными окнами.
* **`ChainBlockDetector.ts`**: Вычисляет цепочки соприкасающихся окон и определяет, есть ли у них физическое пространство для сдвига или сжатия.
* **`ChainTransitions.ts`**: Рассчитывает пропорциональные сдвиги («гармошку»). При расширении одного окна соседи плавно сдвигаются; они начинают сжиматься только тогда, когда цепочка упирается в край экрана, что позволяет сохранять их относительные пропорции.
* **`DragTiling.ts`**: Управляет поведением мыши, включая Drag-and-Drop, жесты обмена (swap) окон местами и журнал транзакций.

## ⚡ Под капотом: Математические алгоритмы (Для разработчиков)

### 1. Эластичный ресайз и сдвиг «гармошкой»
При изменении размеров окон с клавиатуры Dynamic Tiler использует модель эластичной пружины:
* **Сдвиг «гармошкой»:** расширение активного окна плавно расталкивает соприкасающихся соседей в выбранном направлении.
* **Симметричный откат при тупике (Symmetrical Memory Fallbacks):** если окно зажато (например, левые соседи уже упёрлись в край экрана и сжались до своего минимального лимита), нажатие `Super + Left` не заблокирует действие. Вместо этого расширение начнет мягко уменьшать активное окно с *противоположной* стороны, позволяя правому соседу расшириться в освободившееся место. Это предотвращает застревание окон и делает ресайз интуитивно понятным.

### 2. Кратковременная память исправлений (Keyboard Undo)
Если вы случайно изменили размер окна не в ту сторону и сразу же нажали противоположную стрелку, движок расценит это как **жест отмены (Undo)** и вернет окну его точные предыдущие физические размеры.
* Если между нажатиями прошло примерно **2,5 секунды**, противоположная стрелка вернется к обычному поведению (изменению противоположной границы окна).

### 3. Транзакционный Drag-and-Drop и безопасный вынос окон
Перетаскивание окон мышью часто ведет себя непредсказуемо в традиционных тайлерах. Dynamic Tiler решает эту задачу с помощью транзакционного DND-солвера:
* **Безопасное извлечение и откат (Safe Extraction):** когда вы вытаскиваете окно из сетки мышкой, чтобы сделать его плавающим, движок обращается к журналу транзакций (`DragTransactionSnapshot`, где записаны состояния `beforeStates`, `afterStates` и список затронутых окон). Вместо хаотичного растягивания соседей или создания пустых зон, расширение **возвращает затронутых соседей к их точным размерам**, которые были до вставки этого окна. Если за это время раскладка была изменена вручную, система применит стандартное схлопывание вакансии (`collapseVacancy`).

### 4. Детали работы с мышью
* **Позиционирование по курсору:** Целевой слот рассчитывается относительно положения курсора мыши и предпочтительного размера перетаскиваемого окна, а не его верхнего левого угла. Гистерезис памяти сессии сглаживает микроколебания руки, предотвращая раздражающее мерцание превью.
* **Скоринг нарезания (Axis Carving Scoring):** когда вы бросаете окно поверх другого, солвер оценивает горизонтальную и вертикальную оси вставки на основе **минимальной потери полезной площади**. Сброс ближе к верхнему/нижнему краю нарезает сетку вертикально, а к боковым сторонам — горизонтально. Это предотвращает грубое сжатие широких окон (например, браузера) в узкую полоску при случайном дропе терминала рядом с ними.
* **Проскальзывание по коридорам (Corridor Edge Sliding):** если перетащить плавающее окно к самому краю плотной строки, солвер сдвинет всю цепочку окон внутрь, аккуратно сжав широкого соседа, чтобы открыть чистый слот у границы экрана.
* **Явный обмен окон (Explicit Swap Mode):** если во время перетаскивания зажать клавиши обмена (по умолчанию `<Control><Shift>d`), включится режим обмена. Вместо сдвига окон перетаскиваемое и целевое окна просто поменяются местами. Превью при этом сменит цвет на **зеленый**, показывая намерение обмена.
* **Защита от случайного отпускания (Accidental Release Shield):** если вы случайно отпустили клавишу-модификатор в пределах 80 пикселей от начала движения мыши, система заблокирует вынос окна из сетки, предотвращая случайный переход окон во floating-режим при дрожании пальца.

## 🧑‍💻 Контроль качества кода (Для разработчиков)

Dynamic Tiler уделяет большое внимание стабильности геометрических расчетов. Математическое ядро полностью покрыто автоматизированными тестами:

* **Тесты логики (`tests/usecase.test.ts`)**: Проверяют поведение клавиатурных хоткеев, память отмены ресайза, работу кэша и правильность считывания физических размеров.
* **Интеграционные тесты движка (`tests/engine.test.ts`)**: Огромный модуль спецификаций (~100 КБ кода), тестирующий:
  * Пропорциональное сжатие при цепных коллизиях и сдвигах по коридорам.
  * Точность расчета мышиных целей DnD относительно положения курсора.
  * Ведение журналов транзакций и откаты сетки после извлечения окон.
  * Тесты на базе случайных перестановок (property-style fuzzed permutation tests) прогоняют сотни случайных комбинаций расположения и перетаскивания окон, гарантируя отсутствие наложений, выходов за границы экрана и падений движка.

### Полезные команды при разработке:
```bash
# Запуск всех тестов
npm test

# Сборка и автоматическая локальная установка расширения
npm run build:extension

# Сборка CLI-утилиты (dist/cli.js)
npm run build:cli
```

## 📈 Текущий статус и поддержка проекта

Dynamic Tiler в настоящее время находится на **очень ранней стадии развития**. Тем не менее расширение ежедневно и активно используется и тестируется разработчиком в качестве основного инструмента управления окнами в реальной повседневной работе. Это гарантирует постоянное развитие, стабилизацию и планомерное улучшение UX в будущем.

Вы можете напрямую поддержать проект своими **баг-репортами и отзывами**. Ваши отзывы об удобстве использования, пограничных геометрических коллизиях и поведении сетки на разных конфигурациях экранов — это именно то, что делает наш тайлер надежнее и лучше для всех.

Чтобы узнать, как включить логи отладки, собрать лог-файл и правильно составить воспроизводимый баг-репорт, пожалуйста, ознакомьтесь с нашим [Руководством по созданию баг-репортов](CONTRIBUTING.ru.md).

### 🏆 Что уже сделано в последних обновлениях:
* Математика сетки полностью отделена от окружения операционной системы.
